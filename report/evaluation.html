Carregamento dos dados de treinamento e teste¶

Com os dados j tratados e separados pela execuo do pipeline, basta importar os dados necessrios para o teste dos modelos:

  • X_train.pkl: Features j prprocessadas, sem outliers, com dados j normalizados e binarizados (no caso de categorias);
  • balanced_y_train.pkl: Varivel alvo com tratamento de dados desbalanceados;
  • X_test.pkl: Features selecionadas para teste do modelo;
  • y_test.pkl: Varivel alvo, sem tratamento de dados desbalanceados, para teste do modelo.

Avaliação dos Modelos¶

Aqui vamos realizar o treinamento e teste dos modelos citados, com parmetros padres, para avaliar as mtricas de validao de todos. As mtricas utilizadas so:

  • Validao Cruzada;
  • Matriz de Confuso (juntamente com as taxas de verdadeiros positivos, verdadeiros negativos, falsos positivos e falsos negativos);
  • Avaliao da Curva ROC e rea sobre a curva ROC;

E, para o modelo escolhido, ser feito um estudo da acurcia do modelo em funo da complexidade (nmero de features), afim de evitar overfitting. Para este modelo, tambm ser feito uma avaliao da curva de aprendizado.

Gradient Boost Classifier¶

Este classificador tambm um mtodo ensemble, ou seja, utiliza-se da combinao do resultado de um conjunto de classificadores mais simples. Porm, neste caso, o erro de um dos classificadores deste conjunto ajustado pelo classificador anterior. Assim, este um mtodo que apresenta timos resultados (principalmente em competies do Kaggle), principalmente por ser bem robusto contra overfitting.

Escolha dos Hiperparametros do Modelo¶

Aps encontrar o melhor modelo para o problema proposto, vamos selecionar os melhores parametros para o modelo, afim de torna-lo o mais acurado quanto for possvel.

Para isso, utilizamos a biblioteca RandomizedSearchCV, que realiza uma busca aleatria entre os parmetros configurados de modo otimizado, utilizando-se do erro da simulao de um conjunto de parmentros para ajustar a prxima simulao.

In [141]:
from sklearn.model_selection import RandomizedSearchCV
from sklearn.ensemble import GradientBoostingClassifier

model = GradientBoostingClassifier()

parameters = {
    'loss': ['deviance', 'exponential'],
    'learning_rate': [0.01, 0.05, 0.1, 0.5],
    'n_estimators': [10, 30, 50, 100, 150, 200],
    'max_features': [50, 100, 'auto'],
}

clf = RandomizedSearchCV(model, parameters, n_jobs=-1, cv=3)
clf.fit(X_train, y_train)

print('Best parameters found:\n', clf.best_params_)

means = clf.cv_results_['mean_test_score']
stds = clf.cv_results_['std_test_score']
for mean, std, params in zip(means, stds, clf.cv_results_['params']):
    print("%0.3f (+/-%0.03f) for %r" % (mean, std * 2, params))

Outros Detalhes¶

Tratamento de Outliers¶

Para cada caracterstica numrica, foram excludos valores que superam a mdia em mais que trs desvios padres.

Tratamento de Dados Categóricos¶

Foi realizado um encoding de variveis categricas para uma representao binria. Cara categoria foi transcrita para um nmero binrio. Para variveis que apresentam um nmero elevado de categorias (como city), esse processo mostra-se mais robusto contra o problema de dimensionalidade do que outros, tal como o mtodo one-hot encoding.

Geração de novas características a partir de relações¶

Foi utilizado o pacote sklearn.preprocessing.PolynomialFeatures para gerar relaes polinomiais de at segunda ordem entre as caractersticas numricas. Isso contribui para que os algortimos de machine learning encontrem mais facilmente relaes secundrias, no lineares, entre as caractersticas.

Seleção das melhores features para o modelo¶

Durante o pipeline de treinamento, as caractersticas disponveis so avaliadas segundo sua importncia, que determinada pelo valor F da anlise de varincia (F-Value of ANOVA).

O nome das variveis e sua importncia so exportadas para serem utilizadas no processo de treinamento.
